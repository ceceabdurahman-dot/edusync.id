Multi-tenant Strategy
Layer 1 PostgreSQL RLS
Setiap query otomatis difilter oleh policy database. Bahkan jika app lupa WHERE school_id, RLS memblokir akses lintas tenant.
Layer 2 App-level Filter
Middleware ORM inject WHERE school_id=? ke setiap query. Defense-in-depth — jika RLS dimatikan sementara, app masih aman.
Layer 3 JWT Validation
school_id dari JWT divalidasi di middleware sebelum request masuk ke router. Tidak bisa dimanipulasi dari frontend.
Optional PII Encryption
Kolom sensitif (NIK, nomor HP) dienkripsi via pgcrypto. Key di Vault / K8s Secret. Decrypt hanya di application layer.

-- Jan 2025 DROP TABLE IF EXISTS audit_event_y2025m02; -- Feb 2025 -- ══════════════════════════════════════════════ -- AUTO-CREATE partisi bulan berikutnya (run monthly) -- ══════════════════════════════════════════════ CREATE OR REPLACE FUNCTION create_next_audit_partition() RETURNS VOID AS $$ DECLARE next_month DATE := date_trunc('month', now() + INTERVAL '1 month'); part_name TEXT := 'audit_event_y' || to_char(next_month, 'YYYYmMM'); from_val TEXT := to_char(next_month, 'YYYY-MM-DD'); to_val TEXT := to_char(next_month + INTERVAL '1 month', 'YYYY-MM-DD'); BEGIN EXECUTE format( 'CREATE TABLE IF NOT EXISTS %I PARTITION OF audit_event FOR VALUES FROM (%L) TO (%L)', part_name, from_val, to_val ); END; $$ LANGUAGE plpgsql;

-- true = transaction-local -- ══════════════════════════════════════════════ -- Aktifkan RLS pada semua tabel bisnis -- ══════════════════════════════════════════════ ALTER TABLE integration_connection ENABLE ROW LEVEL SECURITY; -- Policy untuk audit_event (read-only untuk non-super-admin) CREATE POLICY audit_event_tenant_read ON audit_event AS PERMISSIVE FOR SELECT TO app_user USING (school_id = current_setting('app.school_id', true)::UUID); -- Super admin bypass RLS (platform ops) CREATE POLICY audit_event_super_admin ON audit_event AS PERMISSIVE FOR ALL TO super_admin USING (true); -- full access -- ══════════════════════════════════════════════ -- DB Roles (prinsip least privilege) -- ══════════════════════════════════════════════ CREATE ROLE app_user NOLOGIN; -- role untuk API & Worker CREATE ROLE super_admin NOLOGIN; -- platform ops saja CREATE ROLE readonly_user NOLOGIN; -- untuk reporting/analytics -- Grant permissions GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA PUBLIC TO app_user; GRANT SELECT ON ALL TABLES IN SCHEMA PUBLIC TO readonly_user; GRANT ALL ON ALL TABLES IN SCHEMA PUBLIC TO super_admin;

Key Queries
Fetch pending jobs (Worker — FOR UPDATE SKIP LOCKED)
-- Worker query: ambil batch job queued tanpa blocking worker lain BEGIN; WITH claimed AS ( SELECT id, job_id, school_id, created_at FROM sync_job_run WHERE status = 'queued' ORDER BY created_at ASC LIMIT 10 FOR UPDATE SKIP LOCKED -- kunci baris, skip yang sedang diproses worker lain ) UPDATE sync_job_run SET status = 'running', started_at = now() WHERE id IN (SELECT id FROM claimed) RETURNING id, job_id, school_id; COMMIT;
Create sync job + enqueue run
-- 1. Insert sync_job INSERT INTO sync_job (school_id, connection_id, name, schedule_cron, config) VALUES ($1, $2, $3, $4, $5) RETURNING id, name, status; -- 2. Insert sync_job_run (queued) untuk immediate trigger INSERT INTO sync_job_run (job_id, school_id, triggered_by) VALUES ($job_id, $school_id, 'api') RETURNING id; -- run_id yang dikembalikan ke FE
Query audit events per tenant dengan range
-- Partition pruning otomatis karena ada filter created_at SELECT id, event_type, resource_type, resource_id, actor_id, metadata, ip_address, trace_id, created_at FROM audit_event WHERE school_id = $1 AND created_at BETWEEN $2 AND $3 AND ($4 IS NULL OR event_type = $4) -- optional filter by type ORDER BY created_at DESC LIMIT $5 OFFSET $6; -- pagination
DLQ: fetch reprocessable jobs
SELECT d.id, d.job_id, d.school_id, d.payload, d.error_reason, d.attempts, d.created_at, j.name AS job_name, j.connection_id FROM sync_job_dlq d JOIN sync_job j ON j.id = d.job_id WHERE d.school_id = $1 AND d.is_permanent = false AND ($2 IS NULL OR d.error_reason = $2) ORDER BY d.created_at DESC FOR UPDATE SKIP LOCKED -- jika reprocess concurrent LIMIT 50;
Upsert Patterns
Upsert field mapping (idempotent)

RLS + PgBouncer — Catatan Penting
PentingKarena RLS menggunakan set_config('app.school_id', ...) yang bersifat transaction-local (parameter ke-3 = true), setting ini AMAN dengan transaction pooling — reset otomatis setiap akhir transaksi.
TipPastikan set_config(..., true) (bukan false). Parameter true = transaction-local, false = session-local yang bisa bocor ke request berikutnya di session pooling.
